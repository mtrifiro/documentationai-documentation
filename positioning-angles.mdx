## The synthesis

A single deck, drawn entirely from this set:

1. **Open with #15** — cut through the noise floor before the audience pattern-matches to the wrong category.

2. **Problem with #1 + #7** — almost-right code, named bug list. Concrete.

3. **Frame with #6** — *Most frameworks assume a careful human. Reboot assumes an agent.*

4. **Solution with #16 + #13** — entities and actions. Naive code, correct system. **The AX-differentiating slide.**

5. **Conviction with #14** — OS, database, browser, Reboot.

6. **Buyer with #22 + #23** — eng leaders who can't review what their team ships.

7. **Vision with #30** — solo founders ship fintech.

8. **Defense (against AX specifically) with revised #33** — programming model, not orchestrator.

The other 30 angles are not waste. They are the **sales toolkit** — the right framing for the CISO call (#29), the platform-engineering call (#3), the DDD-literate backend lead (#34), the agent-builder partner (#25), the procurement conversation (#37), the credibility line in the demo (#32).

## The rule

The angles that earn their slot share a property: they make a **claim about the layer Reboot occupies** that an orchestrator cannot make. Programming model. Entities and actions. Naive code. By construction. Locally correct, globally wrong. These are language-design claims, not infrastructure claims.

The angles that don't earn their slot make claims any infra company can make: *new category, runtime for the agent era, makes bad code safe, enforces correctness.* True for AX. True for Temporal. True for whatever Anthropic ships next quarter.

**The test:** if AX's README could honestly include the sentence, cut it from yours.
